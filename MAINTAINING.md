# Maintaining the Agent Database

This document explains how to add, update, and manage entries in the interactive AI agent database. The system is designed so that **all changes flow from a single file** -- no code modifications required.

---

## Architecture Overview

```
data/agents.json          <-- Single source of truth. Edit THIS file.
       |
       +--> index.html     (interactive web UI, reads JSON at runtime)
       |
       +--> README.md      (tables regenerated by sync script)
```

- **`data/agents.json`** is the canonical database. Every agent lives here.
- **`index.html`** dynamically reads the JSON on page load. All filters, stats, and cards are built from whatever is in the file -- no hardcoded lists.
- **`scripts/sync_readme.py`** regenerates the README.md markdown tables from the JSON so the GitHub repo page stays current.

## Adding a New Agent

### Step 1: Edit `data/agents.json`

Open the file and append a new JSON object to the array. Use the next available `id`.

```json
{
  "id": 144,
  "name": "My New Agent",
  "industry": "Finance",
  "industryGroup": "Finance",
  "description": "A short description of what this agent does.",
  "url": "https://github.com/owner/repo",
  "urlType": "github",
  "framework": "Mastra",
  "category": "",
  "tags": ["finance", "trading", "automation"]
}
```

### Step 2: Sync the README

```bash
python scripts/sync_readme.py
```

This regenerates the table sections in README.md between the `<!-- AGENT_TABLE_START -->` and `<!-- AGENT_TABLE_END -->` markers.

### Step 3: Commit and Push

```bash
git add data/agents.json README.md
git commit -m "feat: add My New Agent (Mastra)"
git push
```

GitHub Pages automatically redeploys. The interactive page will show the new agent with no further action.

## Field Reference

Every entry in `agents.json` uses this schema:

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `id` | integer | Yes | Unique sequential identifier. Use `max(existing ids) + 1`. |
| `name` | string | Yes | Display name of the agent. Keep it concise. |
| `industry` | string | Yes | Specific industry label shown on the card (e.g., "Finance", "AI / Workflow Orchestration"). |
| `industryGroup` | string | Yes | Normalized group for the filter dropdown. Pick from the existing list or create a new one if none fit. |
| `description` | string | Yes | One or two sentences explaining what the agent does. |
| `url` | string | Yes | Link to the source code, notebook, or documentation. |
| `urlType` | string | Yes | Either `"github"` or `"notebook"`. Controls the icon shown on the card. |
| `framework` | string | Yes | The agent framework. See the framework list below. Use the exact string. |
| `category` | string | No | Sub-category within a framework (e.g., "Tools", "Agentic RAG"). Leave as `""` if not applicable. |
| `tags` | string[] | Yes | Keywords for search. Lowercase, hyphenated. 3-5 tags recommended. |

## Adding Mastra Agents

Mastra is a TypeScript-first agent framework. When adding Mastra agents, follow this template:

```json
{
  "id": 144,
  "name": "Mastra Weather Agent",
  "industry": "AI Integration",
  "industryGroup": "AI & Machine Learning",
  "description": "A Mastra agent that fetches weather data and provides natural language forecasts using tool-calling.",
  "url": "https://github.com/mastra-ai/mastra/tree/main/examples/weather-agent",
  "urlType": "github",
  "framework": "Mastra",
  "category": "",
  "tags": ["mastra", "weather", "tool-calling", "typescript"]
}
```

Guidelines specific to Mastra:
- `framework` must be exactly `"Mastra"` (capital M)
- `urlType` is typically `"github"` since Mastra examples are code repos, not notebooks
- Include `"mastra"` in the tags array for searchability
- Use categories if Mastra develops themed sections (e.g., `"Tool Use"`, `"RAG"`, `"Multi-Agent"`)

## Bulk Adding Agents

When adding many agents at once (e.g., importing a full framework's examples):

1. Determine the current max `id`:

```bash
python -c "import json; d=json.load(open('data/agents.json')); print(max(a['id'] for a in d))"
```

2. Prepare your entries starting from `max_id + 1`.

3. You can append them with a script or manually. Validate the JSON afterward:

```bash
python -c "import json; json.load(open('data/agents.json')); print('Valid JSON')"
```

4. Run the sync:

```bash
python scripts/sync_readme.py
```

## Updating an Existing Agent

Find the agent by `id` or `name` in `data/agents.json` and edit the fields directly. Then run the sync script.

Common updates:
- **URL changed** -- Update the `url` field
- **Description improved** -- Update `description`
- **Reclassify industry** -- Update both `industry` and `industryGroup`
- **Add tags** -- Append to the `tags` array

---

## Removing an Agent

Delete the JSON object from `data/agents.json` and run the sync script. Do not reuse the `id` -- just leave the gap.

## Validating the Database

Run these checks before committing:

```bash
# Verify valid JSON
python -c "import json; json.load(open('data/agents.json')); print('OK')"

# Count agents and list frameworks
python -c "
import json
d = json.load(open('data/agents.json'))
fws = sorted(set(a['framework'] for a in d))
print(f'{len(d)} agents across {len(fws)} frameworks: {fws}')
"

# Check for duplicate IDs
python -c "
import json
d = json.load(open('data/agents.json'))
ids = [a['id'] for a in d]
dupes = [i for i in ids if ids.count(i) > 1]
print('Duplicate IDs:', set(dupes) if dupes else 'None')
"

# Sync README
python scripts/sync_readme.py
```

## How the Interactive Page Works

`index.html` fetches `data/agents.json` on page load and:

1. Extracts all unique `framework` values to build filter pills
2. Extracts all unique `industryGroup` values to populate the dropdown
3. Extracts all unique `category` values for the category dropdown
4. Computes live stats (total agents, frameworks, industries)
5. Renders cards with search, filtering, and animations

**Nothing is hardcoded.** Adding a new framework, industry, or category to the JSON automatically creates the corresponding UI elements on the next page load. The framework pills are color-coded from a cycling palette that accommodates any number of frameworks.

## Troubleshooting

| Problem | Solution |
|---------|----------|
| New agent doesn't appear on the page | Make sure `data/agents.json` was committed and pushed. Check for JSON syntax errors. |
| README tables not updating | Verify the `<!-- AGENT_TABLE_START -->` and `<!-- AGENT_TABLE_END -->` markers exist in README.md. |
| JSON parse error | Run `python -c "import json; json.load(open('data/agents.json'))"` to find syntax errors. Common issues: trailing commas, missing quotes. |
| New framework has no color | It will get one automatically from the palette. No action needed. |
| Filters don't show new industry group | Check that `industryGroup` is spelled consistently with existing entries. The dropdown is built from exact string matches. |
